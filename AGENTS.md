Agent 行为约束与输出守则 (Anti-Laziness Protocol) ⚠️
作为 AI Agent，你必须严格遵守以下输出原则。这是衡量你工作质量的核心标准：
1. 禁止一切占位符 (No Placeholders)： 严禁在代码中使用 // ... existing code, // TODO: implement this, /* rest of the function */ 等任何形式的省略注释。你必须始终提供完整、可用、可直接复制/运行的代码。
2. 完整输出强制性： 当要求修改一个组件或文件时，不要只输出修改的那几行片段，必须结合上下文提供修改后完整的文件代码块。
3. 优先使用文件写入技能 (Tool Priority)： 如果你具备文件编辑/写入 (File I/O) 技能，请静默修改目标文件。不要在对话框中用 Markdown 打印长篇大论的代码来浪费 Token 和时间。
4. 终端自检与自我修复 (Self-Correction)： 每次完成重要功能或文件修改后，你必须主动调用 Terminal 技能执行 npx expo export:web (或对应的 Linter 命令)。如果出现错误，请自行阅读报错并修复代码，直到终端 100% 编译成功且无报错，再向我汇报任务完成。
5. 单任务聚焦： 保持专注，一次只解决一个核心模块（例如：只专注写完 WebRTCManager.js），并在完成并测试通过后，主动询问下一步指示。不要试图在一次回复中写完整个游戏。
6. 界面层级约束 (UI Z-Index Rule)：出牌或选中纸牌时，纸牌被遮挡，请确保选中的纸牌或拖拽中的纸牌肯定在最上层 (例如提高 zIndex)，避免被其他纸牌遮挡。不仅拖拽纸牌本身需要有极高的 `zIndex`（例如 999），其所有的父容器（如 `ScrollView`、手牌容器区域等）也必须具有比兄弟组件（如 `zIndex: 10` 的对手卡牌）更高的 `zIndex`，并配合 `overflow: 'visible'` 使用，以防止纸牌在跨区域拖动时被遮挡或裁切。
